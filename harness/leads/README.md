# `harness/leads/` — Per-Lead Memory

The durable, file-based memory for every lead the Comeketo Agent works.
**The source of truth** for the agent's understanding of a lead's history,
profile, and current cycle position.

Code and memory live on the same branch (`main`). The runtime reads these
files via the GitHub Contents API (Octokit) — `.vercelignore` excludes
`harness/` from the deploy bundle, and Vercel's "Ignored Build Step" skips
rebuilds when only `harness/**` changed.

---

## Directory shape

```
harness/leads/
  active/
    {lead_id}__{slug}/        ← one folder per Andre-owned, in-progress lead
      00_meta.json
      01_comms_digest.md
      01b_comms_verbatim.md
      04_profile.md
      05_seven_day_plan.md
      06_discovery.md
      09_andre_alerts.md
      10_andre_feedback.md
      client_ledger.md
      comms/
        call_{YYYY-MM-DD}_{shortid}.json
        email_{YYYY-MM-DD}_{shortid}.json
        sms_{YYYY-MM-DD}_{shortid}.json
        thread_{YYYY-MM-DD}_{shortid}.json
        meeting_{YYYY-MM-DD}_{shortid}.json
  archive/
    {lead_id}__{slug}/        ← moved here when status flips to Won/Lost/Not-Interested
```

Numbered file prefixes (`00_`, `01_`, `01b_`, `04_`, `05_`, `06_`, `09_`,
`10_`) force a stable read-order when an LLM ingests the folder
alphabetically: meta first, comms next, profile/plan/discovery, then
alerts and operator feedback. **Do not renumber.**

---

## File responsibilities

| File | Source of truth | Regenerated by | Hand-edited? |
|---|---|---|---|
| `00_meta.json` | Sweeper | Sweeper | No |
| `01_comms_digest.md` | Sweeper (narrative summary) | Sweeper | No |
| `01b_comms_verbatim.md` | Sweeper (full text) | Sweeper | No |
| `04_profile.md` | LLM regen | `lead-folder-llm.ts` | No |
| `05_seven_day_plan.md` | Supabase `plans` table (mirrored) | Plan-write hook | No |
| `06_discovery.md` | LLM regen | `lead-folder-llm.ts` | No |
| `09_andre_alerts.md` | Sweeper or operator | Sweeper / operator | Yes (operator can add) |
| `10_andre_feedback.md` | **Operator only** | Never overwritten by sweeper | **Yes — this is the override channel** |
| `client_ledger.md` | Sweeper | Sweeper | No |
| `comms/*.json` | Close REST API | Sweeper | No |

The sweeper **never touches `10_andre_feedback.md`**. That file is Andre's
override channel — anything written there beats whatever the sweeper or LLM
produced. The next plan/touch composer reads it before drafting.

## `00_meta.json` shape

```json
{
  "lead_id": "lead_…",
  "name": "Eliana Lopes",
  "slug": "eliana-lopes",
  "contact_id": "cont_…",
  "primary_phone": "+15089678185",
  "primary_email": "eliana.lopes8@icloud.com",
  "altitude": "mid",
  "status_label": "🔴 Lost",
  "event_date": "2026-06-20",
  "plan_day_1_date": "2026-04-27",
  "active_cadence": true,
  "cadence_started_at": "2026-04-27T15:07:00Z",
  "comms_dirty": false,
  "last_sweep_at": "2026-05-05T14:00:00Z",
  "comms_content_hash": "sha256:…"
}
```

`comms_dirty` is flipped to `true` by the Close webhook handler when an
activity event arrives. The sweeper prioritizes dirty leads on its next
pass and flips the flag back to `false` after a successful regen.

`comms_content_hash` lets the LLM regen path skip work when the verbatim
content hasn't changed materially since the last `04_profile.md` /
`06_discovery.md` write.

## How files get written

1. **Hourly cron** (`/api/cron/sweep-leads`) calls `sweepActiveLeads()`.
2. For each Andre-owned, in-progress lead: hydrate via direct Close REST
   (`closeListActivities` + per-call `closeGetCall` for transcripts), render
   to file map, diff against current branch state, write only changed
   files in **one commit per lead**.
3. **Close webhooks** flip `comms_dirty: true` so the next sweep prioritizes
   that lead.
4. **Manual trigger** lives on `/test` for "I just hung up the phone, refresh
   this lead now."
5. **Status transition to Won/Lost/Not-Interested**: one final sweep, then
   `mv` from `active/` to `archive/`.

---

## How files get read

The app on `main` calls helpers in `src/lib/lead-folder.ts`:

```ts
const profile = await readLeadFile(leadId, "04_profile.md");
const verbatim = await readLeadFile(leadId, "01b_comms_verbatim.md");
```

Reads target the configured branch (default `main`). There is a small request-scoped
cache so a single page load doesn't re-fetch the same file multiple times.

## Slug rules

- Lowercase, ASCII, hyphens for spaces, drop punctuation.
- `Eliana Lopes` → `eliana-lopes`
- `Sakamoto Family` → `sakamoto-family`
- `☼ Sunny ☼` → `sunny`
- On collision (rare), append the first 6 chars of the lead_id hash.

The `{lead_id}__{slug}` folder name keeps the lead_id authoritative
(slugs can collide; ids cannot) while making the tree browseable.
